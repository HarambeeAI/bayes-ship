# Minimalist Skill — Editorial UI Aesthetic

## When to use

When the product is content-heavy or tool-focused (project management, note-taking, dashboards with lots of data). The goal is maximum clarity with minimum visual noise.

## Visual characteristics

- **Palette**: Monochrome + one accent color. Gray scale: #fafafa, #f3f3f3, #e5e5e5, #a3a3a3, #525252, #171717.
- **Accent**: Single strong color for primary actions and active states. Blue or violet work well.
- **Typography**: System font stack or a single sans-serif (Inter, SF Pro). Strong size hierarchy.
- **Borders**: Crisp 1px borders (#e5e5e5). Used consistently to separate sections.
- **Border radius**: Small (4-6px) or none. Sharp corners communicate precision.
- **Shadows**: None or very minimal. Elevation through borders, not shadows.

## Layout

- Dense but organized. Efficient use of space.
- Sidebar + main content layout (Notion/Linear pattern)
- Tables are first-class citizens: properly aligned columns, sticky headers, row actions.
- Lists use horizontal rules as separators, not cards.

## Components

- **Navigation**: Sidebar with icon + text. Active state: background highlight, not text color change.
- **Buttons**: Small, text-based. Primary: filled accent. Secondary: text-only or ghost.
- **Inputs**: Minimal border (bottom-only or full, not thick). 36-40px height (compact).
- **Tags/badges**: Pill-shaped, low-contrast background + darker text from same color family.

## This aesthetic fits

- Worryless AI, CogniSocial dashboards
- Internal tools, admin panels
- Data-heavy interfaces
